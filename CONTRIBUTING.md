# Contributing

## Workflow

1. Create a feature branch from `main`.
2. Keep changes scoped and commit with clear messages.
3. Open a pull request and request review.
4. Wait for required CI checks and at least one approval.

## Standards

- Keep code and docs aligned.
- Add or update tests for behavioral changes.
- Avoid breaking contract/public API changes without migration notes.

## Security

Never commit secrets, tokens, or private keys. Use environment variables and secret managers.
